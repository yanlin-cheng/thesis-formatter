# AI 修改论文格式系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
高校学生在撰写学位论文时，需要严格按照学校规定的格式模板进行排版。然而，手动调整论文格式耗时耗力，容易出现格式错误。本项目旨在开发一套自动化系统，能够将学生论文内容智能填充到学校官方模板中，自动生成符合格式要求的学位论文。

### 1.2 项目目标
- 自动解析学生论文内容（章节、图片、表格、参考文献等）
- 将解析内容智能匹配到学校模板的对应位置
- 生成完全符合学校格式要求的 Word 文档
- 支持多所高校的不同模板格式

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 论文内容解析
系统需要能够解析学生提交的论文，提取以下内容：
- **封面信息**：论文标题、作者姓名、指导教师、学位类别、提交日期等
- **摘要部分**：中文摘要、英文摘要、中文关键词、英文关键词
- **正文内容**：各章节标题及内容（支持章、节、小节多级结构）
- **图片元素**：图片文件及对应的标题说明
- **表格元素**：表格内容及对应的标题说明
- **参考文献**：参考文献列表
- **致谢部分**：致谢内容
- **附录部分**：附录内容及编号

#### 2.1.2 模板结构解析
系统需要能够解析学校提供的模板文件，识别：
- 模板的整体结构（封面、摘要、目录、正文、参考文献等）
- 各部分的样式定义（字体、字号、行距、段前段后距等）
- 章节编号格式（如"第 1 章"、"1.1"、"1.1.1"等）
- 图片和表格的编号及标题格式

#### 2.1.3 智能内容填充
系统需要将学生论文内容智能填充到模板中：
- **章节匹配**：根据章节编号和标题，将学生论文章节匹配到模板对应位置
- **样式应用**：自动应用模板定义的样式（标题样式、正文样式等）
- **编号处理**：正确处理章节编号、图片编号、表格编号、公式编号
- **缺失处理**：当学生论文缺少某些章节时，在模板中保留占位提示
- **额外章节处理**：当学生论文有模板未定义的章节时，追加到文档末尾

#### 2.1.4 文档生成
系统需要生成符合要求的 Word 文档：
- 完整保留模板的所有格式设置
- 正确应用各级标题样式
- 自动生成目录
- 正确插入图片和表格及其标题
- 正确格式化参考文献
- 支持输出为 .docx 格式

### 2.2 辅助功能

#### 2.2.1 配置管理
- 支持通过配置文件定义不同学校的模板参数
- 支持自定义章节编号格式
- 支持自定义样式映射关系

#### 2.2.2 日志记录
- 记录处理过程中的关键步骤
- 记录匹配成功/失败的情况
- 记录生成的统计信息

#### 2.2.3 错误处理
- 文件不存在时的友好提示
- 解析失败时的错误定位
- 生成失败时的原因说明

---

## 3. 输入输出需求

### 3.1 输入
- **学生论文**：Word 文档格式（.docx），包含完整的论文内容
- **学校模板**：Word 文档格式（.docx），包含格式定义的空白模板
- **配置文件**：YAML 或 JSON 格式，定义模板参数和样式映射

### 3.2 输出
- **格式化论文**：Word 文档格式（.docx），完全符合学校格式要求
- **处理报告**：文本或 JSON 格式，包含处理统计和匹配情况

---

## 4. 章节编号格式需求

系统需要支持多种章节编号格式：

### 4.1 章编号
- 格式示例：`第 1 章`、`第 2 章`
- 或：`第一章 `、` 第二章`

### 4.2 节编号
- 格式示例：`1.1`、`1.2`
- 或：`1.1.1 `、`1.1.2`

### 4.3 小节编号
- 格式示例：`1.1.1`、`1.1.2`
- 或：`(1)`、`(2)`

### 4.4 编号识别
系统需要能够：
- 从标题文本中正确提取章节编号
- 处理编号与标题之间的空格问题
- 处理中文数字和阿拉伯数字的转换

---

## 5. 样式需求

### 5.1 标题样式
- **章标题**：黑体、小三号、居中、段前段后特定间距
- **节标题**：黑体、四号、左对齐、段前段后特定间距
- **小节标题**：黑体、小四号、左对齐、段前段后特定间距

### 5.2 正文样式
- **正文段落**：宋体、小四号、行距固定值、首行缩进
- **摘要正文**：特定字体字号、特定行距
- **参考文献**：特定字体字号、悬挂缩进

### 5.3 图表样式
- **图片标题**：特定位置（下方）、特定字体字号、居中
- **表格标题**：特定位置（上方）、特定字体字号、居中
- **表格边框**：三线表格式或其他指定格式

---

## 6. 匹配逻辑需求

### 6.1 章节匹配
系统需要实现智能章节匹配：
- **精确匹配**：章节编号完全匹配时，直接填充学生内容
- **部分匹配**：章节编号部分匹配时，进行智能判断
- **未匹配**：模板有但学生没有的章节，显示占位提示
- **额外章节**：学生有但模板没有的章节，追加到文档末尾

### 6.2 匹配容错
- 处理学生论文编号错误的情况
- 处理章节缺失的情况
- 处理章节顺序不一致的情况

---

## 7. 性能需求

### 7.1 处理速度
- 单篇论文处理时间应在合理范围内（建议 5 分钟内）
- 支持批量处理多篇论文

### 7.2 内存使用
- 处理大文件时不应占用过多内存
- 应能处理包含大量图片和表格的论文

### 7.3 稳定性
- 处理过程中不应崩溃
- 异常情况应有适当的错误处理

---

## 8. 兼容性需求

### 8.1 文件格式
- 支持 Word 2007 及以上版本的 .docx 格式
- 输出的文档应能在 Word 中正常打开和编辑

### 8.2 操作系统
- 支持 Windows 系统
- 支持 macOS 系统
- 支持 Linux 系统

### 8.3 Word 兼容性
- 生成的文档应保留所有格式设置
- 生成的文档应能在 Word 中正常编辑和保存
